# Structure Rules

This file defines the required academic structure for generated Arabic legal research papers.

## Required Order

Use this order:

1. Cover
2. Cover copy
3. Quran verse, Hadith, or quotation page
4. Dedication
5. Acknowledgements
6. Methodological introduction
7. Research body
8. Conclusion: summary, results, recommendations
9. References list
10. Appendices if any
11. Table of contents

The table of contents is intentionally placed last because this is the required order for this repository unless a user or institution profile states otherwise.

## Required Introduction Components

The methodological introduction must include:

- تمهيد
- مشكلة الدراسة
- أهمية الدراسة
- الدراسات السابقة
- أهداف الدراسة
- نوع ومنهج الدراسة
- حدود الدراسة
- فروض الدراسة
- تساؤلات الدراسة
- أدوات الدراسة
- مفاهيم الدراسة إن وجدت
- صعوبات الدراسة إن وجدت
- خطة الدراسة

## Body Hierarchy

Use only this hierarchy:

```text
قسم → باب → فصل → مبحث → مطلب
```

The Skill must not use:

- فرع
- بند
- أولًا / ثانيًا
- Numbered heading levels
- Lettered heading levels
- Symbolic heading levels

Use prohibited levels only when the user explicitly asks for them.

## Mبحث Pagination Rule

Each `مبحث` must start on a new page. The `مبحث` title must appear centered at the top of that page in bold.

## Approved Plan Protection

If a user supplies an approved plan:

- Preserve every heading.
- Preserve ordering.
- Preserve names.
- Preserve nesting.
- Do not add new body headings inside the approved hierarchy unless the user asks or the added heading is clearly outside the approved plan, such as cover, conclusion, or references.

## Structure Review Questions

- Does every required front/back matter item exist?
- Are introduction components present?
- Is the body hierarchy restricted to the allowed Arabic levels?
- Does each body heading connect to the research title?
- Does the conclusion contain summary, results, and recommendations?
- Are appendices present only when needed?

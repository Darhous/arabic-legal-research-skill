# Final Review Checklist

Do not claim final success unless each required gate is passed or a limitation is explicitly reported.

## Structure Compliance

- [ ] Required document order is present.
- [ ] Methodological introduction contains all required components.
- [ ] Body hierarchy uses only `قسم → باب → فصل → مبحث → مطلب`.
- [ ] No prohibited heading levels appear unless explicitly requested.
- [ ] Conclusion includes summary, results, and recommendations.
- [ ] References list appears in the required position.
- [ ] Appendices appear only if needed.
- [ ] Table of contents exists.

## Methodology Compliance

- [ ] Title satisfies methodology rules.
- [ ] Research problem is defined.
- [ ] Objectives are clear.
- [ ] Importance and contribution are stated.
- [ ] Study limits are defined.
- [ ] Research method is identified.
- [ ] Results and recommendations derive from the body.

## Plan Compliance

- [ ] If an approved plan exists, it is preserved exactly.
- [ ] No approved heading was changed, reordered, deleted, or renamed.
- [ ] If no approved plan exists, the proposed plan is balanced and connected to the title.

## Citation and Footnote Compliance

- [ ] Direct quotations are quoted and documented.
- [ ] Indirect quotations are rewritten and documented.
- [ ] Quranic quotations include surah and verse.
- [ ] Hadith quotations are verified or marked `Requires Verification`.
- [ ] Internet sources include URL and access date/time.
- [ ] No invented references appear.
- [ ] Every page has at least two relevant footnotes where possible.
- [ ] Footnotes are RTL and right aligned.
- [ ] Per-page footnote restart is validated or reported as unvalidated.

## Formatting Compliance

- [ ] A4 page size.
- [ ] Print-ready margins.
- [ ] Arabic RTL throughout.
- [ ] Arabic academic font.
- [ ] Page borders.
- [ ] Centered bold headings.
- [ ] Each `مبحث` starts on a new page.
- [ ] Justified paragraphs.
- [ ] Paragraph indentation.
- [ ] Controlled line spacing.
- [ ] Footer and footnotes RTL from the right.
- [ ] Page numbers.
- [ ] Bibliography formatting.

## DOCX Readiness

- [ ] DOCX file generated.
- [ ] DOCX opens successfully in Microsoft Word or compatible validator.
- [ ] Table of contents fields are present and updatable.
- [ ] Footnote behavior is checked.
- [ ] Colored and black-and-white versions preserve identical structure.

## Claim Control

- [ ] Delivery claim matches the levels in `rules/output-contract.md`.
- [ ] No print-ready claim appears without DOCX generation, structural validation, and required manual Word validation.
- [ ] No submission-ready claim appears without institution-specific final submission requirements.
- [ ] Any `Requires Verification` marker is disclosed and not hidden.

## Final Result

- [ ] All checks passed.
- [ ] Failed checks are documented.
- [ ] Unverified sources are marked.
- [ ] Final report states the exact delivery claim allowed by `rules/output-contract.md`.

# Workflow: تحليل قضية

## لمن هذا المسار؟

لطالب أو باحث أو محامٍ يريد تحليلًا منظمًا لقضية (وقائع، مسائل قانونية، حجج كل طرف، نتيجة محتملة)
لأغراض أكاديمية أو تحضيرية — **ليس بديلًا عن تحليل قانوني مهني**.

## متى أستخدمه؟

عند دراسة قضية فعلية أو افتراضية ضمن مادة دراسية، أو التحضير لمناقشة/مرافعة، أو تلخيص حكم قضائي.

## خطوات التنفيذ

1. اذكر وقائع القضية بدقة (من، متى، ماذا حدث) — بصياغة عامة إن كانت القضية حقيقية وحساسة.
2. اطلب أولًا "تحديد المسائل القانونية المطروحة" (Issues) قبل أي تحليل.
3. اطلب تحليل حجج كل طرف بشكل منفصل ومتوازن (لا حجة طرف واحد فقط).
4. اطلب ربط كل حجة بالسند القانوني المزعوم، مع تمييز أي سند غير مؤكد بـ `Requires Verification`.
5. اطلب "خلاصة محتملة" لا "حكمًا قاطعًا" — التحليل القانوني الحقيقي يحتاج قاضيًا أو محاميًا مختصًا.

## Prompt جاهز

```text
راجع المستودع التالي والتزم بجميع تعليماته كأنها Skill مفعّلة لديك:
https://github.com/Darhous/arabic-legal-research-skill

لدي قضية أريد تحليلها تحليلًا قانونيًا منظمًا:
الوقائع: [الوقائع بصياغة واضحة]
السؤال المطروح: [إن وُجد سؤال محدد]

أعطني: (1) المسائل القانونية المطروحة، (2) حجج كل طرف بشكل متوازن،
(3) السند القانوني لكل حجة مع تمييز غير المؤكد منها، (4) خلاصة محتملة
غير قاطعة. لا تخترع سوابق قضائية أو نصوصًا قانونية.
```

## أخطاء يجب تجنبها

- ❌ تقديم تحليل يميل لطرف واحد فقط دون عرض حجج الطرف الآخر بنفس الجدية.
- ❌ اعتماد "الخلاصة المحتملة" كحكم قانوني نهائي أو مؤكد.
- ❌ نسخ سوابق قضائية أو نصوص مواد دون التحقق من صحتها من مصدر رسمي.
- ❌ استخدام التحليل في سياق قضائي فعلي دون مراجعة محامٍ مرخّص.

## مخرجات متوقعة

تحليل منظم للمسائل القانونية وحجج الطرفين والسند القانوني لكل حجة، مع خلاصة محتملة غير قاطعة
وقائمة نقاط تحتاج تحققًا أو رأيًا قانونيًا مهنيًا.

# مسار الباحث القانوني

## لمن هذا المسار؟

لباحث قانوني (أكاديمي أو مستقل) يعمل على أبحاث متعددة ويحتاج منهجية ثابتة قابلة للتكرار عبر كل
بحث، وربما تحققًا تنفيذيًا فعليًا عبر بيانات JSON منظمة.

## متى أستخدمه؟

- إعداد أكثر من بحث بنفس المعايير المؤسسية (نفس الترتيب، نفس قواعد التوثيق).
- بناء "حالة بحث" منظمة (Research State) يمكن فحصها آليًا لاحقًا عبر CLI.
- مراجعة اتساق مجموعة أبحاث سابقة وفق نفس المعايير.

## خطوات التنفيذ

1. حدد ملف تعريف مؤسسي مناسب إن وُجد: `profiles/generic-arabic-university/profile.md` أو
   `profiles/police-academy/profile.md`، أو اطلب من النموذج التعامل بمعايير عامة.
2. للاستخدام النصي فقط: استخدم `SKILL.md` عبر شات عادي (راجع [`prompts/general-use.md`](../prompts/general-use.md)).
3. للتحقق التنفيذي الفعلي (موصى به للباحث الجاد): ثبّت المشروع (راجع
   [`docs/quickstart.md`](../docs/quickstart.md) المسار الرابع)، وحوّل بيانات بحثك إلى JSON مطابق
   لـ `schemas/research-state.schema.json`، ثم شغّل:
   ```bash
   legal-research-skill schema-check my-research.json --format json
   legal-research-skill validate my-research.json --fail-on high --format json
   ```
4. راجع أي نتيجة `high` أو `critical` قبل اعتبار بحثك مكتملًا شكليًا.
5. كرر نفس الخطوات لكل بحث جديد لضمان اتساق المعايير عبر كل أعمالك.

## Prompt جاهز (للمرحلة النصية قبل التحقق التنفيذي)

```text
راجع المستودع التالي والتزم بجميع تعليماته كأنها Skill مفعّلة لديك:
https://github.com/Darhous/arabic-legal-research-skill

أنا باحث قانوني أعمل وفق ملف التعريف: [اسم البروفايل أو "عام"].
أعدّ لي هيكل بحث كامل حول: [الموضوع]، مع مقدمة منهجية كاملة العناصر
وحواشي موثقة. اذكر أي عنصر منهجي غير قابل للتطبيق مع سبب واضح.
```

## أخطاء يجب تجنبها

- ❌ تجاهل نتائج `validate` ذات الخطورة `high`/`critical` واعتبار البحث جاهزًا رغم ذلك.
- ❌ خلط بروفايلات مؤسسية مختلفة في نفس البحث دون قصد.
- ❌ الاعتماد فقط على المراجعة النصية دون التحقق التنفيذي إن كنت تنشر أبحاثًا بشكل متكرر ومتسق.
- ❌ تعديل نسخة `schemas/` مباشرة بدل استخدامها كمرجع فحص فقط.

## مخرجات متوقعة

بحث أو مجموعة أبحاث تلتزم بمعايير موحدة قابلة للتحقق، مع تقرير تحقق JSON حتمي (نفس المدخل ينتج
نفس النتيجة دائمًا) يمكن أرشفته كدليل جودة لكل بحث.
